## 需求概述

### 核心需求

1. **客户端无感知** - 使用标准 OpenAI SDK/curl，无需任何改动
2. **同步阻塞** - 客户端死等，直到成功或超时
3. **请求排队** - 支持按客户端请求优先级排队，高优先级优先处理
4. **智能重试** - 节点失败自动切换其他可用节点
5. **超时控制** - 中转站控制超时（如 30min/1h），超时释放连接
6. **完整日志** - 记录每个环节，明确错误归因

### 非需求

- ❌ 不需要异步任务系统（job queue）
- ❌ 不需要客户端轮询
- ❌ 不需要 WebSocket/SSE 推送

## 架构设计

### 整体流程

```
┌─────────────┐
│ 客户端请求   │ (标准 OpenAI API)
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────────────┐
│  HTTP Handler (openai.go)               │
│  - 解析请求                              │
│  - 提取 request priority                 │
│  - 创建 PendingRequest                   │
│  - 入队                                  │
│  - 阻塞等待结果 (带超时)                 │
└──────┬──────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────┐
│  优先级队列 (priority_queue.go)         │
│  - 按 model 分组                         │
│  - 按 request priority 排序              │
│  - 支持懒惰删除 (超时或断开自动跳过)      │
└──────┬──────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────┐
│  Worker 池 (pool.go)                    │
│  - 固定并发数 (如 10)                    │
│  - 从队列取任务                          │
│  - 检查 TPM/RPM 限流                     │
│  - 执行转发 (含重试)                     │
│  - 返回结果                              │
└──────┬──────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────┐
│  智能重试 (forwardWithRetry)            │
│  - 选择节点 (随机选择)                   │
│  - 尝试转发 (proxy.Forward)              │
│  - 失败判断 (isRetryable)                │
│  - 切换节点重试                          │
│  - 记录每次尝试                          │
└──────┬──────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────┐
│  上游节点 (OpenAI/Claude/etc)           │
└─────────────────────────────────────────┘
```

## 核心组件

### 1. 优先级队列 (internal/queue/priority_queue.go)

**职责**:
- 按模型分组管理请求
- 按请求指定的优先级排序
- 支持入队/阻塞式出队
- 线程安全

**数据结构**:
使用 `container/heap` 管理 `PendingRequest`。

### 2. Worker 池 (internal/worker/pool.go)

**职责**:
- 管理固定数量的 worker goroutine
- 执行限流检查
- 调用智能重试逻辑
- 实时转发流式响应

## 配置说明

### config.toml 相关配置

```toml
[worker]
pool_size = 10               # worker 并发数
max_retry_attempts = 3       # 每个请求最多重试次数
retry_delay_ms = 100         # 重试延迟 (毫秒)
max_wait_time_sec = 1800     # 最大排队等待时间

[queue]
default_priority = 0         # 默认优先级
max_queue_size = 10000       # 队列最大长度
```

## 总结

系统采用简化路由逻辑（均等随机选择节点），重点强化请求排队与重试保障。节点级优先级与权重已被移除以降低运维成本。
